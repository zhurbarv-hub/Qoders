# Исправление форматирования формы автобэкапа

**Дата**: 20 декабря 2025  
**Статус**: ✅ ВЫПОЛНЕНО

## Проблема

Форма настройки автоматического бэкапа имела проблемы с форматированием:
- Слишком крупные заголовки полей
- Крупные иконки
- Недостаточный padding в полях ввода
- Отсутствие margin между иконками и текстом
- Нет единообразия в размерах шрифтов

## Решение

### Изменения в `database-management.html`

**1. Уменьшены размеры заголовков полей**
```css
/* Было */
font-weight: 500;

/* Стало */
font-size: 13px;
font-weight: 500;
color: #555;
```

**2. Уменьшены иконки**
```css
/* Было */
font-size: 18px;
vertical-align: middle;

/* Стало */
font-size: 16px;
vertical-align: middle;
margin-right: 4px;
```

**3. Увеличен padding полей ввода**
```css
/* Было */
padding: 8px;

/* Стало */
padding: 10px;
font-size: 14px;
```

## Затронутые поля

Форматирование исправлено для всех полей формы:
- ✅ **Частота** (select)
- ✅ **Время** (time input)
- ✅ **День недели** (select)
- ✅ **День месяца** (number input)
- ✅ **Хранить бэкапы (дней)** (number input)
